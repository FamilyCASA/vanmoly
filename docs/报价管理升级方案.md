# 报价管理升级方案

> 文档版本：V1.0
> 编制日期：2026-05-08
> 项目代号：VANMOLY-V3.2-QM
> 依据来源：用户需求（后台新建报价流程升级）

---

## 一、项目背景

### 1.1 现状分析

当前 VANMOLY-SYS-V3.0 的报价管理模块（QuoteManage.vue）已实现基本的报价功能，但在业务流程、数据结构、用户体验方面存在以下不足：

| 问题 | 影响 | 优先级 |
|------|------|--------|
| 背景图/Logo 上传功能失效 | 无法生成带品牌形象的报价封面 | 高 |
| 物料配置流程与实际业务顺序不一致 | 报价员操作繁琐，易出错 | 高 |
| 物料添加成功但空间未新增行 | 数据结构不一致，汇总/导出出错 | 高 |
| 自定义商品名称显示逻辑混乱 | 物料行信息冗余/缺失 | 中 |
| 物料行缺少工艺增项字段 | 无法处理定制类产品（柜体、门窗等）的特殊工艺费用 | 高 |
| 工艺增项筛选不智能 | 需手动查找，无法直接关联物料库二级分类"特殊工艺" | 中 |
| 缺少门店私库机制 | 门店特有物料无法独立管理，影响公库数据纯净性 | 中 |
| 分类汇总缺失 | 无法按大类核对检查，报价员工作量大 | 中 |
| PDF 签名流程不合理 | 签名应在审核通过后由管理员处理，而非报价员主动签 | 中 |
| 审核通过的 PDF 缺少电子公章 | 报价单不具备法律效力 | 中 |

### 1.2 升级必要性

报价是全案装修服务最核心的成交环节，报价单的质量直接影响客户信任度和成单率。本次升级聚焦以下目标：

1. **流程标准化**：按实际业务顺序重组报价流程（封面→关联→物料→汇总→统计）
2. **数据精准化**：修复物料-空间关联 Bug，完善物料行字段（含工艺增项）
3. **资产安全化**：引入门店私库机制，门店特有物料不影响公库
4. **合规合法化**：PDF 审核流程增加电子公章，提升报价单法律效力

## 二、升级目标

### 2.1 核心目标

| 目标 | 描述 | 量化指标 |

### 5.3 第三步：物料配置 ⭐ 核心改造

#### 5.3.1 新增"导入报价表"按钮

**位置**：物料配置区域顶部工具栏

**功能**：支持 Excel 文件（.xlsx）一键导入，Excel 标准模板包含：

| 列 | 对应字段 |
|----|----------|
| 空间名称 | space_name |
| 物料编号 | sku_code |
| 自定义名称 | custom_name |
| 物料名称 | material_name |
| 规格 | custom_width × custom_depth × custom_height |
| 数量 | quantity |
| 工艺增项 | process_name |
| 工艺数量 | process_quantity |

**处理逻辑**：
1. `openpyxl` 解析 Excel 文件
2. 遍历每一行，按 `sku_code` 匹配公库 `material` 表：
   - **匹配成功** → 引用公库物料数据（单价/单位/图片等）
   - **匹配失败** → 标记为"私库物料"，自动写入 `material_store_private` 表
3. 预览确认界面：显示导入数据列表，标注"新增私库"项
4. 点击"应用"后，私库物料正式存入 `material_store_private`，数据进入物料配置列表

**私库隔离机制**：
- 私库物料 `store_id` 关联当前登录用户的门店
- 查询时优先查公库，公库没有再查当前门店私库
- 私库物料不影响 `sku` 表（公库）的数据

**验收标准**：
- [ ] 导入 100 行 Excel < 3s
- [ ] 私库物料正确写入，store_id 正确
- [ ] 导入后预览界面显示数据条数和来源（公库/私库）
- [ ] 点击"应用"后物料出现在配置列表中

#### 5.3.2 物料行字段详细设计

**完整物料行字段链**：

```
[物料编号 D&B-KT-01]
 + [自定义商品名称（显示在物料名称前）]
 + [筛选物料（MaterialSelector）]

 + [工艺单价]
```

**字段映射表**：

| # | 显示位置 | 字段名 | 数据来源 | 可编辑 |
|---|----------|--------|----------|--------|
| 1 | 物料编号 | sku_code | 公库/私库自动生成 | 只读 |
| 2 | 自定义商品名称 | custom_name | 用户输入 | ✅ |
| 3 | 物料名称 | material_name | 筛选物料或手动输入 | ✅ |
| 4 | 物料主图 | material_image | 引用数据库 URL | 只读 |
| 5 | 单价 | unit_price | 引用公库/私库数据 | 只读 |
| 6 | 单位 | unit | 引用公库/私库数据 | 只读 |
| 7 | 宽度 | custom_width | 用户输入（cm）| ✅ |
| 8 | 深度 | custom_depth | 用户输入（cm）| ✅ |
| 9 | 高度 | custom_height | 用户输入（cm）| ✅ |
| 10 | 定制结果 | custom_result | 公式计算（宽×深×高）| 只读 |
| 11 | 数量 | quantity | 用户输入 | ✅ |
| 12 | 工艺增项 | process_id / process_name | 筛选"特殊工艺"分类 | ✅ |
| 13 | 工艺系数 | process_coefficient | 引用工艺数据，默认 1 | 只读 |
| 14 | 工艺数量 | process_quantity | 用户输入 | ✅ |
| 15 | 工艺单位 | process_unit | 引用工艺数据 | 只读 |
| 16 | 工艺单价 | process_unit_price | 引用工艺数据，默认 0 | 只读 |
| 17 | 工艺金额 | process_amount | 公式计算 | 只读 |
| 18 | 本行金额 | row_total | 定制结果×数量 + 工艺金额 | 只读 |

**自定义商品名称规则**：
- 放在物料名称**前**
- 空间名称**后**不需要商品名称（如："客厅"后直接是物料行，不显示"7门衣柜"作为空间名）
- 示例展示：`D&B-KT-01 + 现代简约玻璃门 + 钢化玻璃中空玻璃门 + [图] + 680 + 元/平方米 + 200×100×50 + 2000000 + 1 + [工艺列]...`

**工艺增项筛选规则**：
- 下拉列表数据来源：查询 `material_category` 表，`parent_name = '特殊工艺'` 的所有二级分类
- 选中工艺增项后，自动带出：`process_unit`（单位）、`process_unit_price`（单价）、`process_coefficient`（系数）
- 无特殊工艺时：`process_coefficient = 1`，`process_unit_price = 0`，`process_quantity = 0`

**筛选物料新增行继承逻辑**：
- 点击"筛选物料"添加新物料行时，**自动继承上一行的 `custom_name`**（自定义商品名称）
- 空间切换后，新建物料行继承当前空间的 `custom_name`

#### 5.3.3 Bug 修复：物料添加成功但空间未新增行

**根因分析**：
当前 `SpaceEditor.vue` 中，添加物料时 `addMaterialChild()` 方法将物料写入 `form.spaces[].children[]`，但 `materialSelector` 组件的确认回调可能只更新了局部状态，未触发父组件的响应式更新。

**修复方案**：
1. `materialSelector` 确认回调统一使用 `emit('confirm', materialData)` 向上传递
2. 父组件 `SpaceEditor.vue` 的 `addMaterialChild` 接收后同时：
   - 更新 `form.spaces[spaceIndex].children[]` 数组（Vue 响应式）
   - 手动触发 `nextTick()` 后验证空间 children 长度 > 0
3. 如果空间 children 长度为 0（数据未写入），强制 `push` 一个包含物料数据的对象
4. 添加日志钩子记录添加操作，便于排查

**验收标准**：
- [ ] 物料选择器确认后，空间内立即显示该物料行
- [ ] 刷新页面后数据持久化（已保存到数据库）
- [ ] 空间内有 N 个物料，children.length === N

### 5.4 第四步：分类汇总

**功能**：自动汇总第三步物料配置清单，按物料大类汇总统计。

**汇总逻辑**：
1. 按 `material_name` 或 `category_id` 聚合
2. 分类层级取物料库一级分类（如：定制柜体、门窗工程、水电材料、涂料地板等）
3. 计算每个大类的：物料种数、小计数量、小计金额

**汇总表格式**：

| 大类 | 物料种数 | 小计数量 | 小计金额 | 占比 |
|------|----------|----------|----------|------|
| 定制柜体 | 12 | 45 | ¥68,500 | 42.3% |
| 门窗工程 | 6 | 18 | ¥22,000 | 13.6% |
| 水电材料 | 8 | 120 | ¥8,800 | 5.4% |
| 涂料地板 | 10 | 350 | ¥18,500 | 11.4% |
| ... | | | | |

**展示位置**：第四步页面区域，支持展开/折叠每个大类明细。

**验收标准**：
- [ ] 分类汇总数据与第三步物料明细数据一致（误差 0）
- [ ] 大类占比之和 = 100%
- [ ] 支持点击大类展开查看明细物料列表

### 5.5 第五步：统计总金额

| 功能 | 说明 |
|------|------|
| 总金额统计 | 汇总所有物料行金额 + 工艺金额 |
| 预览 | 全屏预览报价单最终效果 |
| 保存草稿 | 保存到 `status = draft` |
| 提交审核 | 发送到管理员审核队列，`status = pending` |
| 导出 PDF | 生成报价单 PDF（**取消签名功能**）|

**PDF 导出流程变更**：

```
用户点击"导出PDF"
    ↓
检查 status === 'approved'
    ↓ 是
    ↓ 否（draft/pending/rejected）
┌─────────────────────────────────┐
│ draft/pending: 直接导出，无公章   │
│ rejected: 提示"报价未通过审核"    │
└─────────────────────────────────┘
```

**管理员审核流程**：
1. 管理员在报价管理列表看到 `status = pending` 的报价
2. 点击"审核"打开审核面板
3. 上传电子公章图片（`.png`，透明背景，推荐尺寸 200×200px）
4. 点击"通过"：保存公章图片到 `quote_quote.seal_image`，更新 `status = approved`
5. 审核通过的报价：导出 PDF 时每页右下角自动盖电子公章

**PDF 公章嵌入逻辑**（pdf_generator.py）：
- 加载 `seal_image` 图片
- 每页 PDF 右下角（x: 页面宽度 - 80px, y: 20px）嵌入公章图片
- 公章尺寸：80×80px，带透明度（alpha=0.8）
- 审核通过的报价导出 PDF 时，遍历每一页添加公章

**验收标准**：
- [ ] 总金额实时更新（第三步改动后自动重算）
- [ ] 草稿保存不提交审核
- [ ] 提交审核后状态变为"待审核"，管理员可见
- [ ] 未审核通过的报价导出 PDF 无公章
- [ ] 审核通过的报价导出 PDF 每页右下角有电子公章

## 六、后端 API 设计

### 6.1 接口清单

| 方法 | 路径 | 说明 | 变更 |
|------|------|------|------|
| `GET` | `/api/v3/quotes` | 报价列表 | - |
| `GET` | `/api/v3/quotes/{id}` | 报价详情 | 返回物料行含工艺增项字段 |
| `POST` | `/api/v3/quotes` | 新建报价 | - |
| `PUT` | `/api/v3/quotes/{id}` | 更新报价 | - |
| `DELETE` | `/api/v3/quotes/{id}` | 删除报价 | - |
| `POST` | `/api/v3/quotes/{id}/submit` | 提交审核 | status: draft → pending |
| `POST` | `/api/v3/quotes/{id}/approve` | 管理员审核 | 含 seal_image 上传 |
| `POST` | `/api/v3/quotes/{id}/reject` | 驳回报价 | status: pending → rejected |
| `POST` | `/api/v3/quotes/{id}/export-pdf` | 导出 PDF | 含公章逻辑 |
| `POST` | `/api/v3/quotes/import` | 导入报价表 Excel | 新增，存入私库 |
| `GET` | `/api/v3/materials/private` | 门店私库物料列表 | 新增 |
| `GET` | `/api/v3/material-categories/processes` | 工艺增项分类列表 | 新增（筛选特殊工艺）|

### 6.2 导入报价表接口

**请求**：`POST /api/v3/quotes/import`
**Body**：`multipart/form-data`（file: Excel 文件, store_id: 门店ID）

**处理流程**：
1. 解析 Excel，验证必填列（space_name, material_name, quantity）
2. 遍历行，按 sku_code 匹配公库：
   - 匹配成功：取公库字段
   - 匹配失败：写入 `material_store_private` 表，生成 sku_code（如 `PRIVATE-{store_id}-{序号}`）
3. 返回：`{ imported: N, new_private: M, rows: [...] }`

### 6.3 工艺增项分类接口

**请求**：`GET /api/v3/material-categories/processes`

**返回示例**：
```json
[
  { "id": 101, "name": "烤漆工艺", "coefficient": 1.2, "unit": "米", "unit_price": 50 },
  { "id": 102, "name": "覆膜工艺", "coefficient": 1.1, "unit": "平方米", "unit_price": 80 },
  ...
]
```

> 注：工艺增项数据来源为 `material_category` 表的"特殊工艺"二级分类，上传物料时需按备注上传工艺信息。
