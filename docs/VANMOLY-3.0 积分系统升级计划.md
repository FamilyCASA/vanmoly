# VANMOLY\-3\.0 积分奖励系统升级计划（专供开发者）



**适用范围**：PC 官网 \+ 员工 OA 后台 \+ 员工管理前端
**核心原则**：一人一户、积分永久、审核生效、公开可查、支持混合消费、内部价自动判定、团队团建可申请、数据看板可视化

# 一、系统总体架构（必做）

## 1\.1 用户唯一标识

- 以 **员工登录名** 为唯一积分主体 ID

- 所有积分、消费、扣除、兑换均与登录名绑定

- 客户、线索、报价、合同、服务流程、物料、楼盘均关联操作人登录名

## 1\.2 积分账户结构（固定字段）

每个员工账户永久包含 5 个数字（系统自动计算，不可手动修改）

1. **历史总积分**（永久累计，用于晋升、评级、评优）

2. **可用积分**（可兑换、可消费、可团建抵扣）

3. **已兑换积分**（兑换现金 / 假期）

4. **已消费积分**（购物抵扣，仅记录不可再用）

5. **本月积分**（当月获取，用于排行）

## 1\.3 积分生命周期（底层逻辑）

1. 操作上报 → 审核通过 → 增加 **可用积分** \+ **历史总积分**

2. 兑换奖励 → 扣减 **可用积分** → 增加 **已兑换积分**

3. 购物消费 → 扣减 **可用积分** → 增加 **已消费积分**

4. 团队团建 → 扣减 **可用积分** → 增加 **已兑换积分**

5. **历史总积分只增不减，永久可查**

# 三、积分消费（开发核心逻辑）

## 3\.1 消费统一规则

1. 仅 **可用积分** 可消费

2. 消费后积分进入 **已消费积分**，永久可查

3. 已消费积分 **不可再用于任何兑换 / 抵扣**

4. 历史总积分 **不变化**

## 3\.2 消费模式（支持两种）

1. **纯积分兑换**（积分 ≥ 订单金额）

2. **积分 \+ 现金混合支付**（积分不足部分现金补齐）

## 3\.3 员工内部价自动判定（必做）

1. 员工本人 / 直系亲属购买

2. 单笔订单 **积分支付比例 ≥ 50%**

3. 自动享受：**零售价 5 折（特价除外）**

4. 积分支付比例 ＜ 50%：按正常零售价结算

### 计算公式（开发固定）

**积分支付比例 = 积分抵扣金额 ÷ 订单总金额 × 100%**
**现金支付金额 = 订单金额 − 积分抵扣金额**

## 3\.4 积分消费后账户变化

可用积分 − 消费积分
已消费积分 \+ 消费积分
历史总积分 **不变**

# 四、积分兑换（独立模块：奖励兑换）

## 4\.1 个人兑换

1. **现金兑换**
可用积分按比例兑换现金
扣减：可用积分 → 增加：已兑换积分

2. **带薪休假兑换**
10000 积分 = 5 天带薪假期
每人每年限兑 1 次
扣减可用积分，不影响历史总分

## 4\.2 团队团建兑换（核心复杂模块）

### 基础档位

- 4 人：100000 积分 = 2 天团建 \+ 10000 元基金

- 5 人：120000 积分 = 2 天团建 \+ 11000 元基金

- 6 人：140000 积分 = 2 天团建 \+ 12000 元基金

- 7 人：160000 积分 = 2 天团建 \+ 13000 元基金

- 8 人：180000 积分 = 2 天团建 \+ 14000 元基金

### 计算公式

所需积分 = 100000 \+ \(人数−4\) × 20000
团建基金 = 10000 \+ \(人数−4\) × 1000

### 积分分摊规则（必须实现）

- 负责人扣除积分 = 普通成员 × 1\.5（多 50%）

- 同意参与成员：按系数分摊扣除

- 不参与成员：不扣积分、正常考勤

- 有成员不参与：团队打上 **【团队不完整】** 标记

### 团建审核通过标准（4 项）

1. 团建视频剪辑

2. 现场照片

3. 消费清单 \+ 发票

4. 成员匿名评价

### 报销规则

- 可报销上限 = 团建基金 × 1\.5

- 超上限部分：负责人承担 50%，成员均摊 50%

- 不满意评价 ≥ 50%：超支不报销 \+ 负责人 “不信任” 考评 1 次

九、后台管理——员工管理——积分管理（页面设计）

## 9\.1 积分管理页面整体说明

积分管理页面统一入口：后台管理 → 员工管理 → 积分管理，系统根据当前登录账号权限，自动展示对应界面（员工界面/团队负责人界面/超级管理员界面），界面布局统一风格，核心功能按权限分层展示，所有操作均留痕，支持数据导出。

## 9\.2 员工界面（普通员工权限）

### 核心定位

仅查看本人积分相关信息、操作本人积分消费/兑换，无法查看他人及团队积分数据，无管理权限。

### 页面核心模块

- 个人积分概览（顶部卡片）：展示历史总积分、可用积分、已兑换积分、已消费积分、本月积分，数据实时同步，点击各积分项可查看对应明细。

- 积分明细查询（核心模块）：支持按时间（本月/上月/自定义）、积分类型（获取/兑换/消费/团建扣除）、来源模块（获客/转化/成交等）筛选，展示每条积分的时间、模块、动作、分值、审核人、状态，支持导出本人积分明细。

- 积分操作入口（功能区）：
    积分消费：跳转至积分消费中心，支持商品浏览、积分抵扣、订单提交，自动判定内部价。

- 积分兑换：跳转至个人奖励兑换页，支持现金兑换、带薪休假兑换（显示剩余兑换次数）。

- 团建参与投票：仅当所在团队发起团建申请时，显示投票入口（参与/不参与），提交后不可修改。

规则查看：可查看积分获取分值、消费规则、兑换规则、内部价规则，无修改权限。

### 权限限制

仅查看本人数据，仅操作本人积分，无法查看团队、其他员工积分，无审核、修改、统计权限。

## 8\.3 团队负责人界面（团队管理权限）

### 核心定位

管理本团队所有员工积分，发起团队团建申请、审核团队相关积分操作，查看团队积分统计数据，无平台级管理权限。

### 页面核心模块

- 团队积分概览（顶部卡片）：展示本团队总积分、平均积分、本月总积分、本团队内员工排行、团队积分构成（按获客/转化/成交等阶段统计）；同时支持查看本团队在公司所有团队中的积分排行，可查看各竞争对手团队的具体总积分、平均积分，且可查看竞争对手团队内精英员工的具体积分（仅展示积分数据，不展示精英员工个人隐私信息），便于团队负责人对比差距、激励本团队成员，借鉴竞争对手精英员工的积分获取经验，促进团队整体进步。

- 个人积分模块：同普通员工界面的个人积分概览、积分明细、个人积分操作入口，保留本人积分操作权限。

- 团队员工积分管理（核心模块）：
    员工积分列表：展示本团队所有员工的姓名、登录名、历史总积分、可用积分、本月积分、积分排名，支持按积分高低、本月积分筛选，点击员工姓名可查看该员工积分明细（仅查看，不可修改）。

- 积分审核：仅审核本团队员工提交的、需团队负责人确认的积分动作（如团队相关的获客、跟进动作），显示待审核列表（包含员工姓名、积分动作、提交时间、对应分值、关联业务凭证），支持通过/驳回操作；审核通过后，系统自动为对应员工增加可用积分及历史总积分，审核驳回需填写明确驳回原因（必填），审核记录永久可查。系统自动生成审核日志，详细记录审核人、审核时间、审核结果、员工姓名、积分动作、分值变化等全部详情；该系统日志保密级别设为最高等级，需配置定时备份机制（建议每日凌晨自动备份），同时启用交叉核对功能（不同备份文件互相校验），杜绝数据丢失、篡改，确保审核流程可追溯、数据安全可保障。

团队团建管理（专属模块）：
    团建申请：发起团建申请，选择团队人数，系统自动计算所需积分、团建基金、可报销上限，选择团队成员，发起参与投票。

团建进度查看：查看当前及历史团建申请的状态（待投票/待审核/已通过/已完成/未通过），展示参与成员、不参与成员、积分扣除情况、团建资料上传状态。

团建资料上传：团建完成后，上传视频、照片、消费清单\+发票，提交审核。

团建评价查看：查看团队成员匿名评价统计，查看评价详情（匿名展示）。

团队积分统计：展示本团队月度/季度/年度积分趋势图、各阶段积分占比饼图，支持数据导出（仅本团队数据）。

### 权限限制

仅管理本团队员工，仅审核本团队相关积分动作，无法查看其他团队、平台整体积分数据，无法修改积分规则、系统配置。

    团队积分列表：展示全平台所有团队的名称、负责人、团队人数、团队总积分、平均积分、本月积分、排名，支持按积分高低、团队人数筛选，点击团队名称可查看该团队所有员工积分明细、团队积分统计。

团建审核：审核所有团队提交的团建申请、团建资料，查看团建评价、消费清单，判定是否通过报销，处理超支报销相关事宜，记录审核结果。

团队标记管理：查看所有带有【团队不完整】标记的团队，可查看标记原因，标记清除需满足“下次团建全员参与”条件，系统自动清除或管理员手动清除（需填写原因）。

积分规则配置（专属模块）：
    积分分值配置：可查看所有积分动作的固定分值，支持特殊情况下修改分值（修改后需公示，修改记录永久可查）。

兑换规则配置：配置现金兑换比例、带薪休假兑换规则、团队团建积分档位及基金计算公式，修改后实时生效，需留存修改记录。

内部价规则配置：配置员工内部价折扣比例（默认5折）、特价产品范围，修改后自动同步至积分消费中心。

风控与异议处理：
    积分异常监控：查看积分异常操作记录（如虚假上报、刷分），可扣除异常积分、取消相关员工积分权限，记录处理结果。

积分异议处理：查看员工提交的积分异议申请，核实相关积分记录、审核记录，给出处理意见（维持原判/调整积分），处理结果同步至员工账户。

操作日志查看：查看所有员工、团队负责人、管理员的积分相关操作日志，不可删除、不可修改，支持按操作人、操作类型、时间筛选。

平台积分统计：展示全平台月度/季度/年度积分趋势图、各岗位积分占比、各团队积分对比图、积分消费/兑换统计，支持全平台数据导出。

### 权限说明

拥有全平台积分管理、规则配置、审核、风控、数据统计及导出权限，所有操作均需留痕，确保可追溯、可监督。

## 8\.5 页面通用要求

- 所有界面均需包含“规则说明”入口，方便不同权限用户查看对应规则。

- 数据实时同步，积分变化、审核结果、投票结果、团建状态需实时更新。

- 支持数据导出（Excel格式），导出内容需包含完整的积分相关信息，符合各权限的数据范围。

- 界面操作简洁，核心功能突出，避免冗余，符合OA后台整体操作风格，降低操作成本。

- 所有表单提交、操作确认均需弹出提示，避免误操作；异常情况（如积分不足、审核驳回）需给出明确提示。
