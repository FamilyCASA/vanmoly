## 三、数据库设计

### 3.1 新增表结构

#### 3.1.1 案例展示表 (case_study)

```sql
CREATE TABLE case_study (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title VARCHAR(200) NOT NULL COMMENT '案例标题',
    type VARCHAR(20) DEFAULT '实景' COMMENT '案例类型:实景/在建/设计/视频/图文',
    style VARCHAR(50) COMMENT '风格标签',
    space_type VARCHAR(50) COMMENT '空间类型',
    budget_range VARCHAR(50) COMMENT '预算区间',
    area DECIMAL(10,2) COMMENT '面积(㎡)',
    cover_image VARCHAR(500) COMMENT '封面图URL',
    location VARCHAR(200) COMMENT '小区/楼盘名称',
    customer_name VARCHAR(100) COMMENT '客户称呼(脱敏)',
    description TEXT COMMENT '案例简介',
    tags TEXT COMMENT '标签JSON数组',
    is_featured BOOLEAN DEFAULT 0 COMMENT '是否精选',
    view_count INTEGER DEFAULT 0 COMMENT '浏览次数',
    likes INTEGER DEFAULT 0 COMMENT '点赞数',
    status VARCHAR(20) DEFAULT '草稿' COMMENT '状态:草稿/已发布/已下线',
    tenant_id VARCHAR(20) COMMENT '租户ID',
    created_by INTEGER COMMENT '创建人ID',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_case_type ON case_study(type);
CREATE INDEX idx_case_style ON case_study(style);
CREATE INDEX idx_case_status ON case_study(status);
CREATE INDEX idx_case_featured ON case_study(is_featured);
CREATE INDEX idx_case_tenant ON case_study(tenant_id);
```

#### 3.1.2 案例媒体表 (case_media)

```sql
CREATE TABLE case_media (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    case_id INTEGER NOT NULL COMMENT '关联案例ID',
    media_type VARCHAR(20) COMMENT '媒体类型:image/video/360',
    url VARCHAR(500) NOT NULL COMMENT '文件URL',
    thumbnail VARCHAR(500) COMMENT '视频封面/缩略图',
    sort_order INTEGER DEFAULT 0 COMMENT '排序',
    description VARCHAR(500) COMMENT '说明',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (case_id) REFERENCES case_study(id) ON DELETE CASCADE
);

CREATE INDEX idx_media_case ON case_media(case_id);
CREATE INDEX idx_media_type ON case_media(media_type);
```

#### 3.1.3 留资线索表 (lead)

```sql
CREATE TABLE lead (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name VARCHAR(50) COMMENT '客户姓名',
    phone VARCHAR(20) UNIQUE NOT NULL COMMENT '手机号',
    source VARCHAR(50) COMMENT '来源:案例留资/报价留资/预约量尺/活动参与',
    source_id INTEGER COMMENT '来源关联ID',
    source_page VARCHAR(200) COMMENT '来源页面URL',
    intention TEXT COMMENT '客户意向描述',
    budget VARCHAR(50) COMMENT '预算区间',
    house_type VARCHAR(50) COMMENT '户型',
    area VARCHAR(50) COMMENT '面积',
    status VARCHAR(20) DEFAULT '新线索' COMMENT '状态:新线索/已联系/已到店/已成交/无效',
    assigned_to INTEGER COMMENT '分配员工ID',
    follow_count INTEGER DEFAULT 0 COMMENT '跟进次数',
    first_contact_at DATETIME COMMENT '首次联系时间',
    last_follow_at DATETIME COMMENT '最后跟进时间',
    remark TEXT COMMENT '备注',
    ip_address VARCHAR(50) COMMENT '访客IP',
    user_agent TEXT COMMENT '访客UA',
    tenant_id VARCHAR(20) COMMENT '租户ID',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (assigned_to) REFERENCES employee(id)
);

CREATE INDEX idx_lead_phone ON lead(phone);
CREATE INDEX idx_lead_status ON lead(status);
CREATE INDEX idx_lead_source ON lead(source);
CREATE INDEX idx_lead_assigned ON lead(assigned_to);
CREATE INDEX idx_lead_tenant ON lead(tenant_id);
CREATE INDEX idx_lead_created ON lead(created_at);
```

#### 3.1.4 线索跟进记录表 (lead_follow)

```sql
CREATE TABLE lead_follow (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    lead_id INTEGER NOT NULL COMMENT '关联线索ID',

```

## 四、API 接口规范

### 4.1 通用规范

**请求格式**:
```json
{
  "Content-Type": "application/json",
  "Authorization": "Bearer {token}",
  "X-Tenant-ID": "{tenant_id}"
}
```

**响应格式**:
```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": 1714051200
}
```

**错误码**:
| 状态码 | 含义 |
|--------|------|
| 200 | 成功 |
| 400 | 参数错误 |
| 401 | 未授权 |
| 403 | 权限不足 |
| 404 | 资源不存在 |
| 500 | 服务器错误 |

### 4.2 案例模块 API

#### 获取案例列表
```
GET /api/v3/cases
```

**参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | string | 否 | 类型筛选 |
| style | string | 否 | 风格筛选 |
| keyword | string | 否 | 关键词搜索 |
| is_featured | bool | 否 | 是否精选 |
| page | int | 否 | 页码，默认1 |
| page_size | int | 否 | 每页数量，默认10 |

**响应**:
```json
{
  "code": 200,
  "data": {
    "items": [...],
    "total": 100,
    "page": 1,
    "page_size": 10
  }
}
```

#### 获取案例详情
```
GET /api/v3/cases/{id}
```

**响应**:
```json
{
  "code": 200,
  "data": {
    "id": 1,
    "title": "龙湖天街·现代轻奢",
    "type": "实景",
    "media": [
      {"type": "image", "url": "...", "thumbnail": "..."},
      {"type": "video", "url": "...", "thumbnail": "..."}
    ],
    ...
  }
}
```

#### 创建案例
```
POST /api/v3/cases
```

**请求体**:
```json
{
  "title": "案例标题",
  "type": "实景",
  "style": "现代",
  "cover_image": "https://...",
  "description": "案例简介",
  "tags": ["标签1", "标签2"]
}
```

#### 上传案例媒体
```
POST /api/v3/cases/{id}/media
Content-Type: multipart/form-data
```

**参数**:
| 参数 | 类型 | 说明 |
|------|------|------|
| file | File | 媒体文件 |
| media_type | string | image/video |
| description | string | 说明 |

## 七、开发规范

### 7.1 代码规范

#### Python 后端
- 遵循 PEP 8 规范
- 使用类型注解
- 函数注释使用 Google Style
- 异常必须捕获并记录

```python
def create_case(data: dict, user_id: int) -> Case:
    """
    创建案例
    
    Args:
        data: 案例数据
        user_id: 创建人ID
        
    Returns:
        Case: 创建的案例对象
        
    Raises:
        ValueError: 数据验证失败
    """
    try:
        case = Case(**data)
        case.created_by = user_id
        db.session.add(case)
        db.session.commit()
        return case
    except Exception as e:
        db.session.rollback()
        logger.error(f"创建案例失败: {e}")
        raise
```

#### Vue 前端
- 使用 Composition API
- 组件命名使用 PascalCase
-  Props 必须定义类型和默认值
- 使用 `<script setup>` 语法

```vue
<template>
  <div class="case-card">
    <img :src="coverImage" :alt="title" />

const props = defineProps({

  }
})
</script>
```
