## 一、升级概述

本次升级重点改造**视觉素材模块**，将原有的单一效果图图集升级为**6阶段分段上传展示系统**，符合方案汇报PDF的逻辑结构。

### 升级范围

| 模块 | 改动 | 说明 |
|------|------|------|
| 基础信息 | 不改 | 保持现有功能 |
| 英雄主图 | 不改 | 5张空间主图，保持现有功能 |
| 封面图 | 不改 | 保持现有功能 |
| **效果图图集** | **大改** | 分6阶段上传展示（核心升级） |
| 文案内容 | 删除 | 文案已整合到各阶段中 |
| 报价配置 | 扩展 | 新增引用报价表数据功能 |
| 时间轴 | 保留 | 后期升级 |
| 服务流程 | 保留 | 后期升级 |
| 文件管理 | 保留 | 后期升级 |
| 发布设置 | 保留 | 后期升级 |

## 二、视觉素材6阶段升级方案

### 2.1 阶段概览

| 阶段 | 名称 | 上传内容 | 文案内容 | 前台展示方式 |
|------|------|----------|----------|--------------|
| 1 | 户型分析 | 户型图 | 户型分析文案（2000字内） | 图+文排版 |
| 2 | 设计意境 | 意境图（≤10张） | 客户需求文案 | 精品家居杂志风格图文混排 |
| 3 | 平面规划 | 平面规划图 | 规划文案 | 右图左文（家居杂志风格） |
| 4 | 鸟瞰展示 | 鸟瞰图（≤4张） | 无 | 图片展示 |
| 5 | 效果图首页 | 设计意向图（≤10张） | 中英文对照文案 | 标题+图片+双语文案 |
| 6 | 空间效果图 | 按空间上传效果图 | 标题+简介（可排版） | 空间分组展示 |

## 三、数据库模型升级

### 3.1 新增：案例阶段内容表 (case_phases)

```sql
CREATE TABLE case_phases (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    case_id INTEGER NOT NULL,
    phase_number INTEGER NOT NULL,           -- 阶段编号 1-6
    phase_name VARCHAR(50),                  -- 阶段名称
    
    -- 阶段1：户型分析
    layout_images TEXT,                      -- 户型图JSON数组
    layout_analysis TEXT,                    -- 户型分析文案（富文本）
    
    -- 阶段2：设计意境
    mood_images TEXT,                        -- 意境图JSON数组（≤10张）
    mood_text TEXT,                          -- 客户需求文案（富文本）
    
    -- 阶段3：平面规划
    plan_image TEXT,                         -- 平面规划图URL
    plan_text TEXT,                          -- 规划文案（富文本）
    
    -- 阶段4：鸟瞰展示
    birdview_images TEXT,                    -- 鸟瞰图JSON数组（≤4张）
    
    -- 阶段5：效果图首页
    showcase_images TEXT,                    -- 设计意向图JSON数组（≤10张）
    showcase_title1 VARCHAR(200),            -- 一级标题
    showcase_title2 VARCHAR(200),            -- 二级标题
    showcase_text_cn TEXT,                   -- 中文文案
    showcase_text_en TEXT,                   -- 英文文案
    
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    
    FOREIGN KEY (case_id) REFERENCES case_study(id) ON DELETE CASCADE
);
```

### 3.2 新增：空间效果图表 (case_space_renderings)

```sql
CREATE TABLE case_space_renderings (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    case_id INTEGER NOT NULL,
    space_name VARCHAR(100) NOT NULL,        -- 空间名称（引用报价表）
    sort_order INTEGER DEFAULT 0,            -- 空间排序
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    
    FOREIGN KEY (case_id) REFERENCES case_study(id) ON DELETE CASCADE
);
```

### 3.3 新增：效果图详情表 (case_rendering_items)

```sql
CREATE TABLE case_rendering_items (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    space_id INTEGER NOT NULL,               -- 关联空间
    image_url VARCHAR(500) NOT NULL,         -- 图片URL
    title VARCHAR(200),                      -- 标题
    description TEXT,                        -- 简介（富文本）
    sort_order INTEGER DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    
    FOREIGN KEY (space_id) REFERENCES case_space_renderings(id) ON DELETE CASCADE
);
```

### 3.4 扩展：案例主表字段

```sql
-- 新增字段
ALTER TABLE case_study ADD COLUMN quote_id INTEGER;  -- 关联报价表ID
```

## 四、后端 API 设计

### 4.1 阶段内容 API

```python
# 获取所有阶段内容
GET    /api/v3/cases/<id>/phases

# 更新阶段内容（按阶段更新）
PUT    /api/v3/cases/<id>/phases/1    # 更新户型分析
PUT    /api/v3/cases/<id>/phases/2    # 更新设计意境
PUT    /api/v3/cases/<id>/phases/3    # 更新平面规划
PUT    /api/v3/cases/<id>/phases/4    # 更新鸟瞰展示
PUT    /api/v3/cases/<id>/phases/5    # 更新效果图首页
```

### 4.2 空间效果图 API

```python
# 获取空间列表
GET    /api/v3/cases/<id>/spaces

# 添加空间
POST   /api/v3/cases/<id>/spaces
{
    "space_name": "客厅",
    "sort_order": 1
}

# 更新空间
PUT    /api/v3/spaces/<space_id>

# 删除空间
DELETE /api/v3/spaces/<space_id>

# 获取空间下的效果图
GET    /api/v3/spaces/<space_id>/renderings

# 添加效果图
POST   /api/v3/spaces/<space_id>/renderings
{
    "image_url": "...",
    "title": "光影交错的会客空间",
    "description": "<p>大面积落地窗...</p>",
    "sort_order": 1
}

# 更新效果图
PUT    /api/v3/renderings/<rendering_id>

# 删除效果图
DELETE /api/v3/renderings/<rendering_id>

# 批量排序
POST   /api/v3/spaces/reorder
```

### 4.3 报价引用 API

```python
# 获取可引用的报价表列表
GET    /api/v3/quotes/available

# 引用报价表
POST   /api/v3/cases/<id>/quote
{
    "quote_id": 123
}

# 获取引用报价表的汇总数据
GET    /api/v3/cases/<id>/quote/summary
# 返回：大类汇总报价（基础装修、定制家具、活动家具、软装配饰等）
```

### 4.4 空间名称下拉 API

```python
# 获取报价表的空间名称列表
GET    /api/v3/quotes/<quote_id>/space-names
# 返回：["客厅", "餐厅", "主卧", "次卧", "书房", "厨房", "卫生间", "阳台"]
```
