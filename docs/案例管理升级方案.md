# D&B 帝标|设记家案例管理升级方案

**版本**: V3.1.0  
**日期**: 2026-04-26  
**状态**: 开发大纲

---

## 一、升级目标

将现有简易案例管理升级为**全链路案例运营系统**，支持：
- 案例全生命周期管理（创建→编辑→发布→同步→数据分析）
- 多渠道内容分发（小程序/公众号/小红书）
- 客资收集与转化追踪
- 订阅推送与用户运营

### 2.2 案例时间轴表 (case_timeline)

```sql
CREATE TABLE case_timeline (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    case_id INTEGER NOT NULL,
    node_time DATETIME NOT NULL,              -- 时间节点
    title VARCHAR(100),                       -- 节点标题
    content TEXT,                             -- 内容
    media_urls TEXT,                          -- 图片/视频JSON数组
    sort_order INTEGER DEFAULT 0,             -- 排序
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (case_id) REFERENCES case_study(id) ON DELETE CASCADE
);
```

### 2.3 案例文件表 (case_files)

```sql
CREATE TABLE case_files (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    case_id INTEGER NOT NULL,
    file_type VARCHAR(20),                    -- pdf/image/video
    file_name VARCHAR(200),                   -- 文件名
    file_url VARCHAR(500),                    -- 文件URL
    has_watermark BOOLEAN DEFAULT FALSE,      -- 是否有水印
    download_count INTEGER DEFAULT 0,         -- 下载次数
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (case_id) REFERENCES case_study(id) ON DELETE CASCADE
);
```

### 2.4 案例订阅表 (case_subscriptions)

```sql
CREATE TABLE case_subscriptions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    case_id INTEGER NOT NULL,
    user_id INTEGER,                          -- 注册用户ID (可为空)
    openid VARCHAR(100),                      -- 微信openid
    subscribe_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    notify_enabled BOOLEAN DEFAULT TRUE,      -- 是否接收通知
    last_notify_time DATETIME,                -- 最后通知时间
    FOREIGN KEY (case_id) REFERENCES case_study(id) ON DELETE CASCADE
);
```

### 2.5 客资留资表 (case_leads)

```sql
CREATE TABLE case_leads (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    case_id INTEGER NOT NULL,
    name VARCHAR(100),                        -- 姓名
    phone VARCHAR(20),                        -- 手机号
    email VARCHAR(100),                       -- 邮箱
    wechat VARCHAR(100),                      -- 微信号
    source VARCHAR(50),                       -- 来源 (pdf_download/view_detail/consult)
    message TEXT,                             -- 留言内容
    status VARCHAR(20) DEFAULT 'new',         -- 状态 new/contacted/converted
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (case_id) REFERENCES case_study(id) ON DELETE CASCADE
);
```

### 2.6 案例模板表 (case_templates)

```sql
CREATE TABLE case_templates (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    template_name VARCHAR(200),               -- 模板名称
    package_type VARCHAR(50),                 -- 套餐类型
    price_min DECIMAL(12,2),                  -- 价格区间-最小
    price_max DECIMAL(12,2),                  -- 价格区间-最大
    suitable_house_types TEXT,                -- 适用户型JSON
    base_content TEXT,                        -- 基础文案内容JSON
    created_by INTEGER,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 2.7 推送记录表 (case_notifications)

```sql
CREATE TABLE case_notifications (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    case_id INTEGER NOT NULL,
    notify_type VARCHAR(20),                  -- auto/manual
    content TEXT,                             -- 通知内容
    send_time DATETIME,                       -- 发送时间
    send_status VARCHAR(20),                  -- pending/sent/failed
    receiver_count INTEGER,                   -- 接收人数
    success_count INTEGER,                    -- 成功数
    fail_count INTEGER,                       -- 失败数
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (case_id) REFERENCES case_study(id) ON DELETE CASCADE
);
```

### 2.8 操作日志表 (case_operation_logs)

```sql
CREATE TABLE case_operation_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    case_id INTEGER,
    operator_id INTEGER,                      -- 操作人ID
    operator_name VARCHAR(100),               -- 操作人姓名
    operation VARCHAR(50),                    -- 操作类型
    content TEXT,                             -- 操作内容
    ip_address VARCHAR(50),                   -- IP地址
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 三、后端 API 设计

### 3.1 案例管理 API

```python
# 案例列表
GET    /api/v3/cases                    # 获取案例列表 (支持筛选/排序/分页)
POST   /api/v3/cases                    # 创建案例
GET    /api/v3/cases/<id>               # 获取案例详情
PUT    /api/v3/cases/<id>               # 更新案例
DELETE /api/v3/cases/<id>               # 删除案例

# 批量操作
POST   /api/v3/cases/batch              # 批量操作 (上架/下架/删除/同步)

# 发布管理
POST   /api/v3/cases/<id>/publish       # 立即发布
POST   /api/v3/cases/<id>/schedule      # 定时发布
POST   /api/v3/cases/<id>/unpublish     # 下架

# 同步功能
POST   /api/v3/cases/<id>/sync/xiaohongshu  # 同步至小红书
POST   /api/v3/cases/<id>/sync/mp           # 同步至公众号

# 复制/模板
POST   /api/v3/cases/<id>/copy          # 复制案例
POST   /api/v3/cases/<id>/template      # 保存为模板
```

### 3.2 时间轴 API

```python
GET    /api/v3/cases/<id>/timeline      # 获取时间轴
POST   /api/v3/cases/<id>/timeline      # 添加时间节点
PUT    /api/v3/timeline/<node_id>       # 更新节点
DELETE /api/v3/timeline/<node_id>       # 删除节点
POST   /api/v3/timeline/reorder         # 重新排序
```

### 3.3 文件管理 API

```python
GET    /api/v3/cases/<id>/files         # 获取文件列表
POST   /api/v3/cases/<id>/files         # 上传文件
DELETE /api/v3/files/<file_id>          # 删除文件
GET    /api/v3/files/<file_id>/download # 下载文件 (带水印控制)
```

### 3.4 客资管理 API

```python
GET    /api/v3/cases/<id>/leads         # 获取留资列表
GET    /api/v3/case-leads               # 全局留资列表 (带筛选)
POST   /api/v3/case-leads/<id>/contact  # 标记已联系
POST   /api/v3/case-leads/export        # 导出Excel
```

### 3.5 订阅管理 API

```python
POST   /api/v3/cases/<id>/subscribe     # 订阅案例 (前端)
DELETE /api/v3/cases/<id>/subscribe     # 取消订阅 (前端)
GET    /api/v3/cases/<id>/subscribers   # 获取订阅用户列表
POST   /api/v3/cases/<id>/notify        # 发送更新通知
```

### 3.6 模板库 API

```python
GET    /api/v3/case-templates           # 获取模板列表
POST   /api/v3/case-templates           # 创建模板
GET    /api/v3/case-templates/<id>      # 获取模板详情
PUT    /api/v3/case-templates/<id>      # 更新模板
DELETE /api/v3/case-templates/<id>      # 删除模板
POST   /api/v3/case-templates/<id>/use  # 使用模板创建案例
```

### 3.7 数据统计 API

```python
GET    /api/v3/cases/<id>/stats         # 单个案例统计
GET    /api/v3/cases/stats/overview     # 全局案例统计
GET    /api/v3/cases/stats/trend        # 趋势数据
```

## 七、技术要点

### 7.1 富文本编辑器
- 使用 `vue-quill-editor` 或 `wangEditor`
- 支持图文混排、样式调整
- 图片自动上传至OSS

### 7.2 文件水印
- PDF水印: 使用 `PyPDF2` 或 `reportlab`
- 图片水印: 使用 `Pillow`
- 水印内容: 用户ID + 时间戳 + "D&B 帝标|设记家版权所有"

### 7.3 微信推送
- 使用微信服务号模板消息
- 订阅用户批量推送
- 推送状态追踪

### 7.4 数据导出
- Excel导出: 使用 `xlsx` 库
- PDF生成: 使用 `html2pdf.js` (前端) 或 `weasyprint` (后端)

## 九、注意事项

1. **业主授权**: 所有实景案例必须获得业主书面授权后方可发布
2. **隐私保护**: 客户姓名、地址等信息需要脱敏处理
3. **版权保护**: PDF下载必须带水印，防止盗用
4. **推送频率**: 单个案例每周最多推送一次更新通知
5. **数据备份**: 案例删除采用软删除，保留30天后物理删除
