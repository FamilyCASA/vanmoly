> 文档版本：V1.0
> 编制日期：2026-05-08

> 依据文档：《楼盘调查标准化录入开发方案 V1.0》

---

## 一、项目背景

### 1.1 现状分析

成都梵木里家居设计有限公司（VANMOLY）的业务核心围绕楼盘全案装修服务展开。当前系统（VANMOLY-SYS-V3.0）对楼盘信息的管理仅限于**物理属性和合作信息**（楼盘名称、地址、开发商、物业、坐标、合作状态等），缺乏系统化的**营销调研数据**采集能力。

业务团队在实际工作中依赖线下 Excel 表格（"标准化楼盘调研表录入模板.xlsx"）记录楼盘调研信息，存在以下痛点：

- 数据分散存储于个人设备，无法集中管理
- 格式不统一，无法横向对比分析
- 调研数据与楼盘信息相互孤立，无法联动
- 购房人群占比等关键指标缺乏校验机制

### 1.2 升级必要性

| 问题 | 影响 | 解决路径 |
|------|------|----------|
| 调研数据游离于系统外 | 决策无数据支撑，团队协作效率低 | 将调研数据纳入系统管理 |
| 购房人群占比无校验 | 数据失真，分析价值降低 | 前端实时+后端双重校验 |
| Excel 模板分散 | 版本不统一，录入规范难以执行 | 系统化录入表单+数据格式强制 |
| 楼盘信息与调研信息分离 | 查看楼盘需切换多个数据源 | 1:1 关联，一次查询全部呈现 |

---

## 二、升级目标

### 2.1 核心目标

建立标准化的楼盘调研数据采集与管理平台，实现：

1. **数据标准化**：将原有 18 个 Excel 字段固化到系统表单，消除格式混乱
2. **校验自动化**：购房人群占比前后端双重校验，总和偏离 100% 时实时预警
3. **关联一体化**：楼盘物理信息与营销调研数据同屏呈现，业务人员无需切换
4. **协作便捷化**：团队成员共享楼盘调研数据，支持查看与编辑

### 2.2 量化指标

| 指标 | 当前值 | 目标值 |
|------|--------|--------|
| 调研数据系统化率 | 0% | ≥80%（核心楼盘） |
| 购房人群占比校验覆盖率 | 无 | 100%（前后端双重） |
| 单个楼盘调研录入时间 | ~20min（Excel） | ≤10min（系统表单） |
| 调研数据查询响应时间 | 需打开 Excel | <1s（系统查询） |

## 四、数据库设计

### 4.1 新表结构

**表名**：`building_survey`
**关联关系**：1:1 关联 `building` 表（共享 `building_id`）

| 字段分组 | 字段 | 类型 | 说明 |
|----------|------|------|------|
| **项目概况** | delivery_date | DATE | 交付时间 |
| | delivery_count | INTEGER | 交付数量（套） |
| | base_task | VARCHAR(50) | 保底任务 |
| | property_category | VARCHAR(20) | 住宅/商业/商住两用/别墅 |
| **户型分析** | unit_area | VARCHAR(30) | 户型面积（含"平米"） |
| | unit_count | INTEGER | 户型数量 |
| | main_unit_type | VARCHAR(50) | 主力户型 |
| **购房人群** | age_0_18 ~ age_60_plus | INTEGER×5 | 各年龄段占比（存整数，%前端追加） |
| **配套设施** | matching_shops | TEXT | 匹配商家（逗号分隔） |
| | metro_info | VARCHAR(50) | 地铁配套 |
| | park_info | VARCHAR(50) | 公园配套 |
| **录入信息** | entry_date | DATE | 录入日期 |
| | entry_by | VARCHAR(50) | 录入人 |

### 4.2 购房人群占比校验规则

- 前端：实时计算 5 项之和，超出 100% 或偏差 >5% 时红色警告
- 后端：提交时再次校验，超出容差范围返回 400 错误
- 存储：整数百分比（不含 % 符号），前端渲染时自动追加 %

┌──────────────────────────────────────────────────────────────┐
│  楼盘名称（只读）                                             │

- 偏差 >5%：红色警告"合计：X%，请检查修正"

## 十一、后续规划（Phase 5）

本次升级暂不实施批量导入功能（Phase 5），该功能将作为独立迭代纳入后续排期：

- **功能描述**：支持 Excel 文件上传，自动解析"标准化楼盘调研表录入模板.xlsx"，批量写入数据库
- **技术方案**：`el-upload` + `openpyxl` 解析，错误行收集后统一展示
- **优先级**：低（单条录入可满足初期数据采集需求）
