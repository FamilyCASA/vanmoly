# 楼盘调查标准化录入开发方案

> 文档版本：V1.0
> 创建日期：2026-05-08
> 依据：标准化楼盘调研表录入模板.xlsx

---

## 一、需求分析

### 1.1 模板字段拆解

| 分类 | 字段 | 录入格式 | 数据类型 | 说明 |
|------|------|----------|----------|------|
| **项目概况** | 项目名称 | 文本 | 字符串 | 楼盘完整名称 |
| | 交付时间 | YYYY-MM-DD | 日期 | 预计或实际交付日期 |
| | 交付数量 | 整数 | 数值 | 交付套数（单位：套） |
| | 项目地址 | 文本 | 字符串 | 含省市区街道门牌 |
| | 保底任务 | 整数/文本 | 混合 | 销售任务填金额，其他填说明 |
| | 项目类别 | 选项 | 枚举 | 住宅/商业/商住两用/别墅 |
| **户型分析** | 户型面积 | 数值+平米 | 字符串 | 保留1位小数，如"95.5平米" |
| | 户型数量 | 整数 | 数值 | 户型种类数量 |
| | 主力户型 | 文本 | 字符串 | 主要户型结构，如"三室两厅" |
| **购房人群** | 0-18岁 | 百分比 | 数值 | 购房人群占比，%可省略 |
| | 19-30岁 | 百分比 | 数值 | 同上 |
| | 31-45岁 | 百分比 | 数值 | 同上 |
| | 46-60岁 | 百分比 | 数值 | 同上 |
| | 60岁以上 | 百分比 | 数值 | 同上 |
| **配套设施** | 匹配商家 | 文本 | 字符串 | 逗号分隔多个类型 |
| | 地铁配套 | 文本 | 字符串 | "有（300米）"或"无" |
| | 公园配套 | 文本 | 字符串 | "有"或"无" |

### 1.2 与现有 Building 模型的关系

现有 `Building` 模型侧重于**楼盘物理属性和合作信息**（开发商、物业、坐标、合作状态等）。

新增的楼盘调查表侧重于**营销调研数据**（户型分析、人群画像、周边配套）。两者为 1:1 关联，共享 `building_id`。

## 二、数据库设计

### 2.1 新建 `building_survey` 表

```sql
CREATE TABLE building_survey (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    building_id     INTEGER NOT NULL UNIQUE,
    tenant_id       VARCHAR(32) DEFAULT '0',

    -- 项目概况（除 name/address 已由 Building 表提供）
    delivery_date   DATE        COMMENT '交付时间 YYYY-MM-DD',
    delivery_count  INTEGER     COMMENT '交付数量（套）',
    base_task       VARCHAR(50) COMMENT '保底任务（金额或说明）',
    property_category VARCHAR(20) COMMENT '项目类别：residential/commercial/mixed/villa',

    -- 户型分析
    unit_area       VARCHAR(30) COMMENT '户型面积，如"95.5平米"',
    unit_count      INTEGER     COMMENT '户型数量',
    main_unit_type  VARCHAR(50) COMMENT '主力户型，如"三室两厅"',

    -- 购房人群占比（百分比，整数存储，渲染时自动加%）
    age_0_18        INTEGER     COMMENT '0-18岁购房占比',
    age_19_30       INTEGER     COMMENT '19-30岁购房占比',
    age_31_45       INTEGER     COMMENT '31-45岁购房占比',
    age_46_60       INTEGER     COMMENT '46-60岁购房占比',
    age_60_plus     INTEGER     COMMENT '60岁以上购房占比',

    -- 配套设施
    matching_shops  TEXT        COMMENT '匹配商家，多个逗号分隔',
    metro_info      VARCHAR(50) COMMENT '地铁配套，如"有（300米）"或"无"',
    park_info       VARCHAR(50) COMMENT '公园配套，如"有"或"无"',

    -- 录入信息
    entry_date      DATE        COMMENT '录入日期',
    entry_by        VARCHAR(50) COMMENT '录入人',

    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP,

    FOREIGN KEY (building_id) REFERENCES building(id)
);
```

```python
from datetime import datetime
from app import db


class BuildingSurvey(db.Model):
    """楼盘调查表"""
    __tablename__ = 'building_survey'

    id = db.Column(db.Integer, primary_key=True, autoincrement=True)
    building_id = db.Column(db.Integer, db.ForeignKey('building.id'),
                            nullable=False, unique=True)
    tenant_id = db.Column(db.String(32), default='0', index=True)

    # 项目概况
    delivery_date = db.Column(db.Date, comment='交付时间')
    delivery_count = db.Column(db.Integer, comment='交付数量')
    base_task = db.Column(db.String(50), comment='保底任务')
    property_category = db.Column(db.String(20), comment='项目类别')

    # 户型分析
    unit_area = db.Column(db.String(30), comment='户型面积')
    unit_count = db.Column(db.Integer, comment='户型数量')
    main_unit_type = db.Column(db.String(50), comment='主力户型')

    # 购房人群占比（存储整数，%可省略）
    age_0_18 = db.Column(db.Integer, comment='0-18岁占比')
    age_19_30 = db.Column(db.Integer, comment='19-30岁占比')
    age_31_45 = db.Column(db.Integer, comment='31-45岁占比')
    age_46_60 = db.Column(db.Integer, comment='46-60岁占比')
    age_60_plus = db.Column(db.Integer, comment='60岁以上占比')

    # 配套设施
    matching_shops = db.Column(db.Text, comment='匹配商家')
    metro_info = db.Column(db.String(50), comment='地铁配套')
    park_info = db.Column(db.String(50), comment='公园配套')

    # 录入信息
    entry_date = db.Column(db.Date, comment='录入日期')
    entry_by = db.Column(db.String(50), comment='录入人')

    created_at = db.Column(db.DateTime, default=datetime.utcnow)
    updated_at = db.Column(db.DateTime, default=datetime.utcnow,
                           onupdate=datetime.utcnow)

    def to_dict(self):
        return {
            'id': self.id,
            'building_id': self.building_id,
            'delivery_date': self.delivery_date.isoformat() if self.delivery_date else None,
            'delivery_count': self.delivery_count,
            'base_task': self.base_task,
            'property_category': self.property_category,
            'unit_area': self.unit_area,
            'unit_count': self.unit_count,
            'main_unit_type': self.main_unit_type,
            'age_0_18': self.age_0_18,
            'age_19_30': self.age_19_30,
            'age_31_45': self.age_31_45,
            'age_46_60': self.age_46_60,
            'age_60_plus': self.age_60_plus,
            'matching_shops': self.matching_shops,
            'metro_info': self.metro_info,
            'park_info': self.park_info,
            'entry_date': self.entry_date.isoformat() if self.entry_date else None,
            'entry_by': self.entry_by,
            'created_at': self.created_at.isoformat() if self.created_at else None,
        }

    # 购房人群占比数组（前端渲染用）
    def get_age_groups(self):
        return [
            {'label': '0-18岁', 'value': self.age_0_18, 'key': 'age_0_18'},
            {'label': '19-30岁', 'value': self.age_19_30, 'key': 'age_19_30'},
            {'label': '31-45岁', 'value': self.age_31_45, 'key': 'age_31_45'},
            {'label': '46-60岁', 'value': self.age_46_60, 'key': 'age_46_60'},
            {'label': '60岁以上', 'value': self.age_60_plus, 'key': 'age_60_plus'},
```

### 3.2 路由层 (`routes/building_survey_routes.py`)

```python
from flask import Blueprint, request, jsonify
from app import db
from app.models.building_survey import BuildingSurvey
from app.models.building import Building
from app.utils.auth import jwt_required_v2

bp = Blueprint('building_survey', __name__, url_prefix='/api/v3/building-surveys')

PROPERTY_CATEGORIES = ['residential', 'commercial', 'mixed', 'villa']


@bp.route('/<int:building_id>', methods=['GET'])
@jwt_required_v2
def get_survey(building_id):
    """获取指定楼盘的调查表"""
    survey = BuildingSurvey.query.filter_by(building_id=building_id).first()
    if not survey:
        return jsonify({'code': 404, 'msg': '该楼盘尚未录入调查表'}), 404
    return jsonify({'code': 200, 'data': survey.to_dict()})


@bp.route('', methods=['POST'])
@jwt_required_v2
def create_survey():
    """新建楼盘调查表"""
    data = request.get_json()

    building_id = data.get('building_id')
    if not building_id:
        return jsonify({'code': 400, 'msg': 'building_id 不能为空'}), 400

    # 检查楼盘是否存在
    building = Building.query.get(building_id)
    if not building:
        return jsonify({'code': 404, 'msg': '楼盘不存在'}), 404

    # 检查是否已有调查表
    existing = BuildingSurvey.query.filter_by(building_id=building_id).first()
    if existing:
        return jsonify({'code': 400, 'msg': '该楼盘已存在调查表，请使用更新接口'}), 400

    # 校验购房人群占比
    age_fields = ['age_0_18', 'age_19_30', 'age_31_45', 'age_46_60', 'age_60_plus']
    total = sum(int(data.get(f, 0) or 0) for f in age_fields)
    if total > 0 and abs(total - 100) > 5:
        return jsonify({'code': 400, 'msg': f'购房人群占比总和为 {total}%，应接近100%'}), 400

    survey = BuildingSurvey(building_id=building_id)
    _apply_data(survey, data)
    db.session.add(survey)
    db.session.commit()
    return jsonify({'code': 200, 'msg': '创建成功', 'data': survey.to_dict()})


@bp.route('/<int:building_id>', methods=['PUT'])
@jwt_required_v2
def update_survey(building_id):
    """更新楼盘调查表"""
    survey = BuildingSurvey.query.filter_by(building_id=building_id).first()
    if not survey:
        return jsonify({'code': 404, 'msg': '调查表不存在'}), 404

    data = request.get_json()

    # 校验购房人群占比
    age_fields = ['age_0_18', 'age_19_30', 'age_31_45', 'age_46_60', 'age_60_plus']
    total = sum(int(data.get(f, 0) or 0) for f in age_fields)
    if total > 0 and abs(total - 100) > 5:
        return jsonify({'code': 400, 'msg': f'购房人群占比总和为 {total}%，应接近100%'}), 400

    _apply_data(survey, data)
    db.session.commit()
    return jsonify({'code': 200, 'msg': '更新成功', 'data': survey.to_dict()})


@bp.route('/<int:building_id>', methods=['DELETE'])
@jwt_required_v2
def delete_survey(building_id):
    """删除楼盘调查表"""
    survey = BuildingSurvey.query.filter_by(building_id=building_id).first()
    if not survey:
        return jsonify({'code': 404, 'msg': '调查表不存在'}), 404
    db.session.delete(survey)
    db.session.commit()
    return jsonify({'code': 200, 'msg': '删除成功'})


@bp.route('/batch', methods=['POST'])
@jwt_required_v2
def batch_import():
    """批量导入调查表（Excel 上传后解析导入）"""
    # TODO: 支持 Excel 文件上传解析
    return jsonify({'code': 501, 'msg': '批量导入功能开发中'})


def _apply_data(survey, data):
    """将请求数据写入模型字段"""
    date_fields = ['delivery_date', 'entry_date']
    int_fields = ['delivery_count', 'unit_count',
                  'age_0_18', 'age_19_30', 'age_31_45', 'age_46_60', 'age_60_plus']
    str_fields = ['base_task', 'property_category',
                  'unit_area', 'main_unit_type',
                  'matching_shops', 'metro_info', 'park_info', 'entry_by']

    for f in date_fields:
        val = data.get(f)
        if val:
            # 兼容 YYYY-MM 或 YYYY-MM-DD 格式
            from datetime import datetime
            try:
                if len(str(val)) == 7:
                    survey.__setattr__(f, datetime.strptime(val, '%Y-%m').date())
                else:
                    survey.__setattr__(f, datetime.strptime(val, '%Y-%m-%d').date())
            except ValueError:
                pass

    for f in int_fields:
        val = data.get(f)
        if val is not None:
            try:
                survey.__setattr__(f, int(val))
            except (ValueError, TypeError):
                pass

    for f in str_fields:
        survey.__setattr__(f, data.get(f))
```

**左侧导航卡片（4个分组）：**

```
┌─────────────────────┐
│ 📋 项目概况          │  ← active
```

**右侧表单分组（对应左侧4个卡片）：**

#### 分组一：项目概况
| 字段 | 组件 | 说明 |
|------|------|------|
| 交付时间 | `el-date-picker` type="date" format="YYYY-MM-DD" | 支持填写年份即可 |
| 交付数量 | `el-input-number` :min="1" | 单位：套 |

#### 分组二：户型分析
| 字段 | 组件 | 说明 |
|------|------|------|
| 户型面积 | `el-input` | 末尾拼接"平米"字样 |

#### 分组三：购房人群
| 字段 | 组件 | 说明 |

> 表单下方显示提示："占比总和应接近100%，偏差较大时需检查修正"

#### 分组四：配套设施

## 五、数据校验规则

### 5.1 必填项（建议）

| 字段 | 规则 |
|------|------|
| 项目概况 | 至少填写交付时间或项目类别一项 |
| 户型分析 | 至少填写主力户型一项 |
| 购房人群 | 总和接近100%允许±5%偏差，超出提示但不阻止提交 |
| 配套设施 | 全部可选 |

### 5.2 格式规则

| 字段 | 规则 |
|------|------|
| 交付时间 | 兼容 `YYYY-MM-DD` 和 `YYYY-MM` 格式 |
| 交付数量 | 正整数 |
| 保底任务 | 混合类型，金额用"万"为单位（如"500万"）|
| 户型面积 | 末尾固定追加"平米"显示 |
| 年龄占比 | 0-100 整数，%号自动追加不存入DB |
| 匹配商家 | 中文逗号分隔存入Text |

## 八、技术备注

1. **共用水路**：楼盘名称和地址从现有 `Building` 表读取，不在调查表中重复存储
2. **%号处理**：百分比符号在前端表单显示时追加，存入 DB 时只存整数
3. **模板复用**：本方案可直接用于其他行业楼盘调研表的数据标准化
4. **编码注意**：后端文件使用 UTF-8，Windows PowerShell 执行注意 chcp 65001
